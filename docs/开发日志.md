- 文件：`Reverse/output/tweak/bypass.mm` +9行
- 改动：新增 `hook_class_getImageName`（返回 `n ?: ""`），在 `hookAntiDebug()` 中通过 MH 宏安装
- 原因：Qunar v5.3.19 `QNRRenderMonitorHookManager` 的 dispatch_once 块（NSNotificationCenter 触发，启动约28分钟后首次运行）调用 `getsectiondata(__DATA/__objc_classlist)` 遍历 ObjC 类列表，对每个类调用 `class_getImageName` 后直接传入 `strcmp`，无 NULL 检查；某个类（stub class 或 ElleKit trampoline）的 `class_getImageName` 返回 NULL → `strcmp(NULL, ...)` → SIGSEGV。Binary 级证据：v5.3.19 UUID `1c07fc68...` binary 偏移 0x10210EFE8 = `bl _strcmp`，上一条指令 `bl _class_getImageName`（0x10347a904）。对称于 v25 对 `hook_dyld_name` 的修复。
