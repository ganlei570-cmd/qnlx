# 去哪儿旅行 iOS 一键新机 逆向分析报告

## 这个程序/接口是干什么的

去哪儿旅行 iOS App（v5.3.18，`com.qunar.iphoneclient8`）的注册流程：用户输入手机号 → 点"获取验证码" → 服务端发短信验证码。目标是在 Dopamine 越狱设备上绕过 GTS（极验安全）SDK 的越狱检测，使服务端正常发送**短信**验证码（而非语音电话）。

---

## 发现了什么

### 注册验证码请求链

- **入口**：`QSMSCodeLoginVC.getSmsCodeClick` → `fetchSmsCode:` → `sendSMSCode:(param)`
- **网络端点**：`https://slugger.qunar.com/slugger-proxy?qrt=p_ucGetVcodeV2`
- **请求体**：xkjgaol 加密密文，约 3431 字节，非明文 protobuf，手机号不可见

### sendSMSCode: 两条路径

| 参数类型 | 触发条件 | 行为 |
|---------|---------|------|
| `NSString*`（手机号） | GTS 返回 nil token 时 | 无 token 校验 → 直接触发**语音验证** |
| `RiskAndPwdInfoModel*` | GTS 返回正常 model 时 | xkjgaol native 校验 token → 检测到越狱 → 弹窗拦截 |

### xkjgaol 调用栈（v71 实测，语音路径）

```
frame 7: xkjgaol + 8024028   ← 发起方（GTS 内部调度）
frame 6: xkjgaol + 8025152
frame 5: xkjgaol + 8093048
frame 4: xkjgaol + 8095204
frame 3: QunariPhone_Cook_CM + 6096088  ← slugger HTTP client
frame 2: xkjgaol + 8958988   ← 创建 NSURLSession task
frame 1: QunariPhone_Cook_CM + 6201544  ← NSURLSession 内部
frame 0: 我们的 net_capture hook
```

**结论**：整条链在 xkjgaol native C code + libdispatch 后台线程内执行，**无 ObjC 拦截点**。

### GTS SDK 信息

- 符号名：`xkjgaol`，可通过 `dlsym(RTLD_DEFAULT, "xkjgaol")` 解析
- 弹窗函数偏移：xkjgaol + 8968112（`0x108b490` 相对基址）
- ObjC 包装：`GTCDeviceUtils.isJailbreak` / `GTSDeviceUtils.isJailbreak`（已 hook，返回 NO）

### 越狱检测绕过现状

| 检测向量 | 是否绕过 | 方法 |
|---------|---------|------|
| stat/stat64/lstat/lstat64 | ✅ | hook → ENOENT |
| access | ✅ | hook → -1 |
| fopen | ✅ | hook → NULL |
| fstat | ✅ | hook → ENOENT |
| open() / openat() | ❌ 尝试失败 | 系统框架用此调用 /private/preboot，hook 后崩溃 |
| _dyld_image_count/_dyld_get_image_name | ✅ | hook，过滤 jb dylib |
| task_info TASK_DYLD_INFO | ✅ | hook，过滤 jb dylib |
| sysctl PT_DENY_ATTACH | ✅ | hook |
| GTCDeviceUtils.isJailbreak (ObjC) | ✅ | %hook 返回 NO |
| xkjgaol native token 校验 | ❌ 未突破 | 根源：fetchSmsCode: 阶段 token 生成失败（返回 nil） |

## 怎么复现（当前 v73 语音路径）

1. 伴生 App 写入新机 profile（spoof IDFV/IDFA/磁盘）
2. 重启去哪儿 App
3. 进注册页面 → 输入手机号
4. `sendSMSCode:` hook 检测 param=nil + gCachedPhone 有值 → 调 `%orig(gCachedPhone)` → GTS 走语音路径
5. 结果：收到**语音电话**（非短信，目前因中国移动骚扰过滤被拦截）
